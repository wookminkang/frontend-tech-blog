왜 useEffect에서
async 함수를 직접
사용하면 안 되나요?

2026년 4월 14일 4분 읽기 React, useEffect, async/await
!

useEffect의 콜백은 동기 함수이거나, cleanup 함수를 반환해야 합니다. async 함수는 항상 Promise를 반환하므로 useEffect와 직접 함께 사용하면 예상치 못한 메모리 누수와 버그가 발생합니다.

01
어떤 문제가 발생하나요?

useEffect는 콜백 함수가 반환하는 값을 cleanup 함수로 인식합니다. 정상적인 경우 return () => {} 형태의 함수 또는 아무것도 반환하지 않아야 합니다.

async 함수를 직접 전달하면 React는 Promise 객체를 받게 됩니다. React는 이것을 처리하지 못하고, 비동기 작업이 컴포넌트 언마운트 이후에도 상태를 업데이트하려 해 메모리 누수가 발생합니다.

  • 언마운트 후 상태 업데이트 시도로 인한 메모리 누수
  • cleanup 함수가 실행되지 않아 구독 해제, 타이머 해제 불가
  • race condition 발생 가능
  • React 18 StrictMode에서 이중 실행으로 인한 버그 증폭
02
잘못된 사용 (Bad Example)
Bad tsx
123 456 7 
// async 함수를 useEffect에 직접 전달 — 금지
useEffect(async () => {
  const data = await fetchData();
  setData(data);
}, []);
// async 함수는 Promise를 반환
// React가 cleanup 대신 Promise를 받게 됨
async () => {}는 항상 Promise를 반환합니다. React는 cleanup 함수 자리에 Promise를 받아 무시하고, 비동기 작업이 완료된 후 컴포넌트가 이미 언마운트되었어도 setData를 호출해 오류를 일으킵니다.
올바른 방법
03
올바른 사용법 — 내부 함수 선언
Good tsx
123 456 78 
useEffect(() => {
  // 내부에 async 함수를 선언하고 바로 호출
  const load = async () => {
    const data = await fetchData();
    setData(data);
  };
  load();
}, []);
useEffect 콜백 자체는 동기 함수로 유지하고, 내부에 async 함수를 선언해 즉시 호출합니다. 이렇게 하면 useEffect가 cleanup 함수를 올바르게 받을 수 있습니다.
04
메모리 누수 방지 — cleanup 함께 사용

컴포넌트가 언마운트될 때 비동기 작업의 결과를 무시하려면 플래그를 활용하세요.

Best tsx
123 456 789 101112 1314 
useEffect(() => {
  let isMounted = true;

  const load = async () => {
    const data = await fetchData();
    // 언마운트 후에는 상태 업데이트 방지
    if (isMounted) setData(data);
  };

  load();

  // cleanup: 언마운트 시 플래그 해제
  return () => { isMounted = false; };
}, []);
isMounted 플래그로 언마운트 후 상태 업데이트를 차단하고, cleanup 함수에서 플래그를 false로 설정해 메모리 누수를 방지합니다.
정리

useEffect에 async 함수를 직접 전달하면 안 됩니다. 콜백 내부에 async 함수를 선언하고 즉시 호출하는 방식을 사용하세요.

컴포넌트가 언마운트될 가능성이 있다면 isMounted 플래그와 cleanup 함수를 함께 사용해 메모리 누수를 반드시 방지하세요.
