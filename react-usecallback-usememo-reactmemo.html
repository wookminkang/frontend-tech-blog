React에서 useCallback, useMemo, React.memo는 언제 써야 하나요?

핵심 요약

React.memo는 props가 바뀌지 않으면 컴포넌트 리렌더링을 건너뜁니다. useCallback은 함수의 참조를 고정해 불필요한 재생성을 막고, useMemo는 연산 결과값의 참조를 고정합니다. 세 가지 모두 메모이제이션 비용이 있으므로, 실제 성능 문제가 확인된 곳에만 적용해야 합니다.

왜 이런 문제가 발생하나요?

React는 부모 컴포넌트가 리렌더링되면 자식도 기본적으로 함께 리렌더링됩니다. 함수와 객체는 참조 타입이라 부모가 리렌더링될 때마다 새 참조로 만들어집니다. React.memo로 감싼 자식도 props 참조가 바뀌었다고 판단해 리렌더링됩니다.

리렌더링 흐름

부모 리렌더링 → 함수/객체 새 참조 생성 → 자식도 리렌더링
부모 리렌더링 → useCallback / useMemo로 참조 고정 → React.memo가 리렌더링 차단
React.memo

컴포넌트 메모이제이션

컴포넌트를 감싸서 이전 props와 새 props를 얕은 비교합니다. props가 같으면 리렌더링을 건너뜁니다. useCallback, useMemo와 함께 사용할 때 효과가 있습니다.

useCallback

함수 참조 고정

의존성 배열이 바뀌지 않으면 같은 함수 참조를 유지합니다. React.memo로 감싼 컴포넌트에 함수를 props로 내릴 때 함께 사용합니다.

useMemo

연산 결괏값 고정

의존성 배열이 바뀌지 않으면 이전 계산 결과를 재사용합니다. 무거운 연산 결과나 객체/배열을 props로 내릴 때 참조를 고정합니다.

세 가지 모두 메모이제이션된 값을 저장하고 의존성을 비교하는 비용이 있습니다. 단순한 컴포넌트나 빠른 연산에 무분별하게 적용하면 최적화가 아니라 오히려 성능을 저하시킬 수 있습니다.

예시 코드 — 잘못된 사용

Bad — React.memo 없이 useCallback만 사용, 단순 연산에 useMemo 남용
import { useState, useCallback, useMemo } from 'react';

export default function ProductPage() {
  const [count, setCount] = useState(0);

  // 단순 증가 함수에 useCallback을 쓸 필요가 없습니다.
  const increment = useCallback(() => {
    setCount((c) => c + 1);
  }, []);

  // 단순한 문자열 연산에 useMemo를 쓸 필요가 없습니다.
  const label = useMemo(() => {
    return `현재 수량: ${count}`;
  }, [count]);

  const handleSearch = useCallback((value: string) => {
    console.log(value);
  }, []);

  return (
    <div>
      <p>{label}</p>
      <button onClick={increment}>추가</button>
      // React.memo로 감싸지 않으면 useCallback은 아무 효과가 없습니다.
      <SearchBox onSearch={handleSearch} />
    </div>
  );
}

// React.memo 없이 선언 — 부모 리렌더링 시 항상 함께 리렌더링됩니다.
function SearchBox({ onSearch }: { onSearch: (v: string) => void }) {
  return <input onChange={(e) => onSearch(e.target.value)} />;
}

React.memo 없이 useCallback만 사용하면 함수 참조가 고정되어도 자식은 부모 리렌더링 시 함께 리렌더링됩니다. useCallback과 useMemo는 React.memo와 짝을 이룰 때 의미가 생깁니다.

올바른 사용법

Good — React.memo + useCallback 세트, useMemo는 무거운 연산에만
import { useState, useCallback, useMemo, memo } from 'react';

export default function ProductPage() {
  const [count, setCount] = useState(0);
  const [products, setProducts] = useState<Product[]>(initialProducts);
  const [filterText, setFilterText] = useState('');

  // useMemo — 항목이 많은 리스트 필터링처럼 비용이 큰 연산에 사용합니다.
  // filterText나 products가 바뀔 때만 재계산됩니다.
  const filteredProducts = useMemo(
    () => products.filter((p) => p.name.includes(filterText)),
    [products, filterText]
  );

  // useCallback — React.memo로 감싼 자식에 함수 props를 내릴 때 사용합니다.
  // count가 바뀌어도 handleFilter 참조는 유지됩니다.
  const handleFilter = useCallback((text: string) => {
    setFilterText(text);
  }, []);

  return (
    <div>
      <p>총 {count}개 담음</p>
      // handleFilter 참조가 고정되어 있어 FilterInput은 리렌더링되지 않습니다.
      <FilterInput onFilter={handleFilter} />
      <ProductList products={filteredProducts} />
    </div>
  );
}

// React.memo — onFilter 참조가 바뀌지 않으면 리렌더링을 건너뜁니다.
const FilterInput = memo(function FilterInput({
  onFilter,
}: {
  onFilter: (text: string) => void;
}) {
  return <input onChange={(e) => onFilter(e.target.value)} />;
});

// 아이템이 많고 렌더링 비용이 있을 때 React.memo가 효과적입니다.
const ProductList = memo(function ProductList({
  products,
}: {
  products: Product[];
}) {
  return (
    <ul>
      {products.map((p) => (
        <li key={p.id}>{p.name}</li>
      ))}
    </ul>
  );
});

React.memo + useCallback은 세트로 사용합니다. useMemo는 계산 비용이 실제로 큰 연산에만 적용합니다. 단순한 연산이나 자주 바뀌는 값에 적용하면 메모이제이션 비용이 이득보다 커집니다.

정리

최적화 훅을 꺼내기 전에 먼저 아래 순서로 접근하세요.

  • 상태를 실제로 사용하는 컴포넌트 가까이 내리기
    불필요하게 상위에 있는 상태가 자식 전체를 리렌더링시키는 경우를 먼저 제거합니다.
  • children prop으로 렌더링 범위 분리하기
    props.children은 부모 리렌더와 무관하게 bail-out됩니다. 컴포넌트 합성으로 범위를 좁히는 것이 훅보다 우선입니다.
  • React.memo + useCallback / useMemo 적용
    위 두 방법으로 해결이 안 될 때, 실제 성능 문제가 측정된 곳에만 투입합니다.
도구 메모이제이션 대상 짝 적합한 상황
