프론트엔드에 관련한 질문이에요.

무거운 상태 업데이트 때문에 UI가 멈추는 현상, useTransition으로 어떻게 해결하나요?

핵심 요약

useTransition은 React 18에서 도입된 훅으로, 특정 상태 업데이트를 "덜 급한 업데이트"로 표시할 수 있게 해줍니다. React는 이렇게 표시된 업데이트를 잠시 미뤄두고, 버튼 클릭이나 텍스트 입력처럼 사용자가 즉각적인 반응을 기대하는 작업을 먼저 처리합니다. 결과적으로 무거운 렌더링이 진행되는 동안에도 UI가 굳지 않고 반응성을 유지할 수 있습니다.

왜 이런 문제가 발생하나요?

React는 기본적으로 상태가 바뀌면 즉시 렌더링을 시작합니다. 이때 렌더링할 컴포넌트가 많거나 계산이 무거우면, 그 렌더링이 끝날 때까지 다른 어떤 작업도 처리하지 못합니다.

예를 들어 검색창에 글자를 입력할 때마다 10,000개 항목을 필터링해야 한다면, 필터링 렌더링이 완료될 때까지 입력창의 커서조차 움직이지 않는 "UI가 굳는" 현상이 발생합니다.

useTransition은 이 문제를 해결하기 위해 상태 업데이트에 우선순위를 부여합니다.

Urgent — 즉시 처리

급한 업데이트

  • 텍스트 입력창의 글자 반영
  • 버튼 클릭 애니메이션
  • 체크박스 토글
Transition — 여유 있을 때

전환 업데이트

  • 검색 결과 목록 렌더링
  • 탭 전환 콘텐츠 렌더링
  • 페이지 이동 시 데이터 렌더링

startTransition으로 감싼 상태 업데이트는 "전환 업데이트"로 분류되어, 급한 업데이트가 들어오면 잠시 중단되고 급한 업데이트를 먼저 처리합니다.

예시 코드 — 잘못된 사용

Bad — 타이핑할 때마다 10,000개 항목이 즉시 렌더링
import { useState } from 'react';

const ITEMS = Array.from({ length: 10000 }, (_, i) => `항목 ${i + 1}`);

function SlowList({ query }: { query: string }) {
  const filtered = ITEMS.filter((item) => item.includes(query));
  return (
    <ul>
      {filtered.map((item) => (
        <li key={item}>{item}</li>
      ))}
    </ul>
  );
}

export default function SearchPage() {
  const [query, setQuery] = useState('');

  return (
    <>
      <input
        value={query}
        onChange={(e) => setQuery(e.target.value)} // 타이핑 즉시 무거운 렌더링 시작
        placeholder="검색어를 입력하세요"
      />
      <SlowList query={query} />
    </>
  );
}

setQuery가 호출되는 순간 React는 즉시 SlowList 렌더링을 시작합니다. 10,000개 항목 필터링이 끝나기 전까지 입력창도 멈추기 때문에, 빠르게 타이핑하면 글자가 뒤늦게 나타나거나 화면이 버벅이는 현상이 발생합니다.

올바른 사용법

Good — 입력창은 즉시 반응, 리스트 렌더링은 startTransition으로 처리
import { useState, useTransition } from 'react';

const ITEMS = Array.from({ length: 10000 }, (_, i) => `항목 ${i + 1}`);

function SlowList({ query }: { query: string }) {
  const filtered = ITEMS.filter((item) => item.includes(query));
  return (
    <ul>
      {filtered.map((item) => (
        <li key={item}>{item}</li>
      ))}
    </ul>
  );
}

export default function SearchPage() {
  const [inputValue, setInputValue] = useState(''); // 입력창에 즉시 반영
  const [query, setQuery] = useState('');         // 리스트에 전달되는 값
  const [isPending, startTransition] = useTransition();

  function handleChange(e: React.ChangeEvent<HTMLInputElement>) {
    setInputValue(e.target.value); // 급한 업데이트 — 입력창 즉시 반응

    startTransition(() => {
      setQuery(e.target.value); // 전환 업데이트 — 여유 있을 때 처리
    });
  }

  return (
    <>
      <input
        value={inputValue}
        onChange={handleChange}
        placeholder="검색어를 입력하세요"
      />
      {isPending && <p>목록을 불러오는 중...</p>}
      <SlowList query={query} />
    </>
  );
}

상태를 두 개로 분리한 것이 핵심입니다.

  • inputValue: 입력창에 즉시 반영되어야 하는 값 — startTransition 밖에서 업데이트합니다.
  • query: SlowList에 전달되는 값 — startTransition 안에서 업데이트합니다.

isPending은 전환 업데이트가 아직 처리 중일 때 true가 됩니다. 이를 이용해 "목록을 불러오는 중..." 같은 로딩 피드백을 사용자에게 보여줄 수 있습니다.

정리

useTransition은 상태 업데이트에 우선순위를 부여하여 무거운 렌더링 중에도 UI 반응성을 유지하는 React 18의 동시성 훅입니다.

구분 일반 setState startTransition
처리 시점 즉시 동기 렌더링 급한 업데이트 이후 처리
UI 블로킹 발생 방지
진행 여부 확인 불가 isPending으로 확인 가능
주요 사용처 입력창, 버튼 상태 등 검색 결과, 탭 전환, 페이지 이동
